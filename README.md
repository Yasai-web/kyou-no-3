# 「きょうの3つ」アプリ 設計仕様書

このドキュメントは、毎日のタスク・楽しみ・予定をまとめて「見るとわくわくする」シンプルなWebアプリを作るための完全な仕様書です。この一枚を読めば、HTML本体・CSV5枚・GitHub Pages公開まで実装できます。

---

## 0. このアプリの思想（最初に読んでほしいこと）

このアプリの目的は「やることを管理する」ことではなく、**「今日やることが軽やかに絞られて、見るとわくわくする」**ことです。実装中に迷ったら、常に「機能を足す」より「軽やかさを守る」を優先してください。具体的には：

- 一日にメインで取り組むのは **3つだけ**。これ以上を画面の主役にしない。
- 「やらなければ」という義務感より、「これだけでいい」という安心感を優先する。
- 全部終わったら **何も残らない静かな達成画面** になる。システムから「もっとやれ」とは決して言わない。

この思想に反する実装（タスクを増やす方向の自動提案、未達成を強調する赤字や警告、ストリーク途切れの罪悪感を煽る演出など）は入れないでください。

---

## 1. 技術構成

- **単一のHTMLファイル**（`index.html`）に HTML / CSS / JavaScript をすべて含める。ビルド工程なし。
- 外部依存は **CDN経由のみ**：
  - Google Fonts（`Zen Maru Gothic` と `Noto Sans JP`）
  - Tabler Icons（`@tabler/icons-webfont`、`ti ti-*` クラスで使用）
- データは **同じリポジトリに置いた5枚のCSV** を `fetch` で読み込む。SheetJS等のライブラリは不要、軽量な自前CSVパーサで足りる（仕様は後述）。
- 「今日の状態」（完了チェック・交換・選んだ内容・ローテーション履歴）は **localStorage** に保存。
- サーバーサイドなし。GitHub Pages で静的公開。
- **CORS問題は起きない**：CSVはHTMLと同一オリジン（同じリポジトリ）にあるため。
- **localStorage以外のブラウザストレージは使わない**。

---

## 2. データ：CSV 5枚

すべて UTF-8、1行目はヘッダー。リポジトリ直下に置く。アプリは起動時に5枚すべてを読み込む。
**キャッシュ回避**：`fetch` 時に `?t=` + タイムスタンプのクエリを付け、プッシュ後に再読み込みすれば最新が反映されるようにする。

### 2-1. `routines.csv` — 単発で繰り返すルーティン（今日の3つの候補になる）

各行がそれ自体で「今日の3つ」の候補になる単発の行動。

| 列 | 内容 | 例 |
|---|---|---|
| name | タスク名（そのまま表示される） | 朝のストレッチ |
| min | 所要時間（分、数値） | 5 |
| days | 候補に出す曜日。`日月火水木金土` を連記、または `毎日` | 火土 / 毎日 |
| energy | エネルギータグ。`軽い` / `ふつう` / `重い`（空欄は `ふつう` 扱い） | 軽い |

```csv
name,min,days,energy
朝のストレッチ,5,毎日,軽い
ボルダリング,90,火土,重い
日記をひとこと,5,毎日,軽い
```

### 2-2. `projects.csv` — 継続的な取り組み（「次の一手」が今日の3つの候補になる）

プロジェクト本体は大きすぎてタスクにできないので、**`next`（次の一手）列に書いた小さな具体的アクションだけ**が候補になる。曜日の概念はない（プロジェクト作業は曜日に縛られないため）。**このファイルが、ユーザーが日常的に手を入れる唯一の生きたファイル**（次の一手が終わるたびに `next` を書き換える）。

| 列 | 内容 | 例 |
|---|---|---|
| name | プロジェクト名（カードでは小さい補足として表示） | VTuberセットアップ |
| next | 次の一手（カードでは大きい主文として表示） | VMagicMirrorをiPhoneと接続 |
| energy | エネルギータグ。`軽い` / `ふつう` / `重い`（空欄は `ふつう`） | ふつう |

```csv
name,next,energy
VTuberセットアップ,VMagicMirrorをiPhoneと接続,ふつう
YouTube一括更新,スクリプトを10本テスト実行,ふつう
翻訳アプリ,用語集の仕様をメモ,軽い
```

**ルーティンとプロジェクトの違い（実装者向けメモ）**：ルーティンは「名前＝やること」、プロジェクトは「名前＝入れ物、next＝やること」。today画面のカード表示で、ルーティンは name を主文に、プロジェクトは next を主文・name を補足に出す。

### 2-3. `events.csv` — 楽しみなイベント（カウントダウン表示）

| 列 | 内容 | 例 |
|---|---|---|
| name | イベント名 | 8月の旅行 |
| date | 日付（`YYYY-MM-DD`） | 2026-08-01 |

```csv
name,date
8月の旅行,2026-08-01
```

今日以降で最も近い未来のイベント1件を「あと◯日」と表示する。当日は「今日!」。過去のイベントは表示しない。

### 2-4. `fun.csv` — きょうのたのしみ（眺めるだけ・チェックなし）

毎日すべて表示。**完了チェックは付けない**（やってもやらなくてもいい、ただそこにある楽しみ）。

| 列 | 内容 | 例 |
|---|---|---|
| name | 楽しみの名前 | 漫画を読む |

```csv
name
ボルダリング
漫画を読む
〇〇のアニメを見る
```

アイコンは name の内容から自動で割り当てる（後述のアイコン対応表）。

### 2-5. `daily.csv` — まいにち（毎日固定の軽い習慣・チェックあり）

毎日必ず表示される軽い習慣。**完了チェックあり**。**「今日の3つ」の自動選択からは除外**（別枠で常に出るため）。少数に絞るのが望ましい。

| 列 | 内容 | 例 |
|---|---|---|
| name | タスク名 | 日記をひとこと |

```csv
name
朝のストレッチ
日記をひとこと
```

---

## 3. 「一日」の定義とリセット（重要）

- **一日の境界は早朝4:00**。深夜に作業していても4時までは「まだ今日」として扱う。
- 実装：「現在時刻から4時間引いた日付」をその日のキー（`dayKey`）とする。
  例：6/14 の 1:30 → 4時間引くと 6/13 21:30 → `dayKey = 2026-06-13`。6/14 の 4:30 → `dayKey = 2026-06-14`。
- localStorage に保存した状態に `dayKey` を持たせ、起動時に現在の `dayKey` と異なれば**その日の状態をすべてリセット**（今日の3つの選択、全完了チェック、まいにちのチェックが白紙に戻り、朝の画面①から始まる）。
- `lastPicked`（ローテーション履歴）は **リセットしない**（日をまたいで蓄積し、最近選ばれていないものを優先するために使う）。

## 4. 画面と遷移

全6画面。スマホ縦（幅360〜430px想定）のモバイルファースト。1カラム。

```
①朝：今日は、どう決める？
   ├─「じぶんで選ぶ」→ ②候補リスト（最大3つタップ）→ ③へ
   └─「気分に合わせる」→ 軽め/ふつう/しっかり の3択 → 自動選択 → ③へ
③今日はこれだけ（日中のホーム）
   ├─ タスクの交換アイコン → ④個別交換（じぶんで選ぶ／気分に合わせる で1枠だけ差し替え）→ ③へ
   ├─「残りを選び直す」→ ⑤選び直し（完了分ロック）→ ③へ
   └─ 今日の3つが全部完了 → ⑥今日はおしまい
⑥今日はおしまい
   └─「もう1つやる？」→ ②と同じ候補リスト → 4枚目を追加 → ⑥へ戻る
```

### ① 朝：今日は、どう決める？
- 日付ラベル＋見出し「今日は、どう決める？」
- 2つのモードカード：
  - **じぶんで選ぶ**（青系アイコン `ti-hand-finger`）：「リストから気分でタップ。最大3つ。」
  - **気分に合わせる**（柑子系アイコン `ti-cloud`）：「今日のエネルギーに合う3つを自動で。」タップで下に3択（軽めの日 `ti-feather` / ふつう `ti-walk` / しっかり `ti-flame`）が展開。
- 日付が変わると必ずこの画面から始まる。

### ② じぶんで選ぶ（候補リスト）
- 候補を2グループで表示：「ルーティン」（routines.csv）と「プロジェクトの次の一手」（projects.csv の next）。
- 各候補はタップで選択トグル。**最大3つまで**（3つ選ぶと他は選べない、または4つ目で一番古い選択を外す等は実装者判断、シンプルに「3つで打ち止め」でよい）。
- ルーティンで「今日が実施曜日」のものには `今日` バッジを付ける。
- 各候補にエネルギータグを補足表示（`5分 · 軽い` など。プロジェクトは `プロジェクト名 · ふつう`）。
- 確定ボタン：「この◯つでいく」（選択数を反映）。0個のときは無効＋「タップして選ぶ（1〜3つ）」。
- 「もどる」リンクで①へ。

### ③ 今日はこれだけ（日中のホーム）
画面の構成（上から順に）：
1. 日付ラベル＋見出し「今日はこれだけ」
2. **今日の3つ**（小見出し「今日の3つ」）：選んだタスクのカード。
   - 未完了：丸い空チェック＋主文＋補足（`ルーティン · 90分 · 重い` / `VTuberセットアップ · ふつう`）＋右端に交換アイコン `ti-switch-horizontal`（控えめな灰色）。
   - 完了：緑チェック＋打ち消し線＋補足「完了!」＋カード全体を少し沈める（opacity 0.75）。完了カードには交換アイコンを出さない。
   - タップで完了トグル。
3. **きょうのたのしみ**（小見出し「きょうのたのしみ」）：fun.csv をピル（角丸999px）で横並び。アイコン＋名前のみ。**チェックなし・タップ不可**。
4. **まいにち**（小見出し「まいにち」）：daily.csv を2列の小さめカードで表示。丸チェック＋名前。タップで完了トグル（完了で緑チェック＋打ち消し線）。
5. **カウントダウン**：柑子色の帯。`ti-plane-departure` ＋「◯◯まで」＋「あと◯日」。
6. 「残りを選び直す」リンク → ⑤へ。

### ④ 個別交換
- ③で未完了タスクの交換アイコンを押したとき。**1枠だけ**を入れかえる画面。
- 上部に「いま入っているもの」（破線の枠で対象タスクを表示）。
- 下に「かわりに入れる」として、①朝と同じ**2つのモードカード**を出す（UXを統一）：
  - **じぶんで選ぶ**（`ti-hand-finger`）：タップすると候補リスト（②と同じ候補だが、**今日すでに選ばれているもの・完了したものは除外**）を表示。1つタップで差し替わって③に戻る。リストからは「もどる」でカード選択に戻れる。
  - **気分に合わせる**（`ti-cloud`）：タップで下に3択（軽め `ti-feather` / ふつう `ti-walk` / しっかり `ti-flame`）が展開。気分を1つ選ぶと、**第5章のロジックでその枠に合う1つだけを自動選択**して差し替わり、③に戻る。母集合は今日すでに選ばれている3つを除外するので、**必ず別の項目に変わる**。
- 選んだ気分に合う候補が0件のときは、「この気分に合う候補がありません」と画面内に表示し、カード選択のまま留まる（別の気分や「じぶんで選ぶ」を選び直せる）。
- 「やめておく」リンクで③へ。

### ⑤ 残りを選び直す
- ③で「残りを選び直す」を押したとき。
- 完了済みのタスクは**ロックして上に残す**（緑チェック＋打ち消し線＋鍵アイコン `ti-lock`＋「完了! このまま残ります」）。
- 空いた枠の数（例：残り2つ）だけ、モードを選び直す：「じぶんで選ぶ」「気分に合わせる」の2カード。見出しは「残り◯つは、どう決める？」。
- 選び直しの候補からは**今日すでに選ばれたもの・完了したものを除外**。
- 「もどる」リンクで③へ。

### ⑥ 今日はおしまい
- 今日の3つが**全部完了**したら③から自動で切り替わる。
- 構成（上から）：
  1. 日付ラベル＋見出し「今日はおしまい」
  2. **今日の3つ**：完了カードを③と**同じ様式**で表示（緑チェック・打ち消し線・「完了!」・opacity 0.75）。※③とカード様式を完全に統一すること。
  3. **きょうのたのしみ**：③と同じピル（チェックなし）。
  4. **言葉だけの締め**：「3つぜんぶ、よくやりました」（丸ゴシック太字）＋「あとは自由時間です」（小さく）。※装飾の大きな丸（太陽）は置かない。
  5. **まいにち**：③と同じ様式。
  6. **カウントダウン**：③と同じ。
  7. 隅に小さく「もう1つやる？」リンク（控えめな下線リンク）。主役はあくまで「おしまい」の締め。
- 「もう1つやる？」→ ②と同じ候補リスト（見出しだけ「もう1つ、なににする？」、1つだけ選ぶ）→ 選んだ1つが4枚目のカードとして③形式で加わる → 終わればまた⑥へ。何周でもできるが、システムからは促さない。

## 5. 「気分に合わせる」の選択ロジック

3択それぞれで、候補の母集合を絞ってから3つを選ぶ。

- **軽めの日**：energy が `軽い` のものだけから選ぶ。
- **ふつう**：energy が `軽い` または `ふつう` のものから選ぶ。
- **しっかりやれる日**：全部から選ぶ。可能なら `重い` を1つ優先的に含める。

母集合の中での3つの選び方：
1. ルーティンは「今日が実施曜日（days に該当）」のものを優先。
2. ローテーション：`lastPicked`（最後に選ばれた dayKey）が**古いもの／一度もないものを優先**して、いつも同じものばかりにならないようにする。
3. ルーティンとプロジェクトの次の一手を**バランス良く**混ぜる（例：ルーティン最大2＋プロジェクト1。母集合次第で調整）。
4. 合計3つ。母集合が3つ未満ならある分だけ。

選ばれたものは `lastPicked = 今日のdayKey` を記録（localStorage）。
**daily.csv（まいにち）はこの自動選択の母集合に含めない**。

## 6. localStorage のデータ構造（提案）

キー名は実装者判断でよいが、最低限以下を保持：

```json
{
  "dayKey": "2026-06-13",
  "todayPicks": [
    { "source": "routine|project", "name": "...", "next": "...", "done": false }
  ],
  "dailyDone": { "朝のストレッチ": true, "日記をひとこと": false },
  "lastPicked": { "朝のストレッチ": "2026-06-13", "VTuberセットアップ::VMagicMirrorをiPhoneと接続": "2026-06-10" }
}
```

- `todayPicks` は「今日の3つ」（＋「もう1つやる？」で増えた分）。順序と完了状態を保持。
- `lastPicked` のキーは、ルーティンは name、プロジェクトは `name::next` のように一意化する（next が変わったら別物として扱われてよい）。
- 起動時：保存された `dayKey` ≠ 現在の `dayKey` なら、`todayPicks` と `dailyDone` を空にして①から。`lastPicked` は保持。
- **CSVが読み込めなかった場合**（オフライン等）：前回読み込めた内容があればそれで動作を継続する（CSVの最終読み込み結果も localStorage にキャッシュしておくとよい）。読み込めず キャッシュもなければ、その旨の静かな案内を出す。

## 7. デザイン

「朝の霧」をイメージした、淡く静かなトーン。

### カラーパレット
| 用途 | 色 |
|---|---|
| 背景（外） | `#F3F6F6` |
| カード背景 | `#FFFFFF` |
| 文字（主） | `#2A3439` |
| 文字（補足・グレー） | `#7E8C94` |
| 文字（淡い・プレースホルダ） | `#AEB9BE` |
| 罫線 | `#E3E9EA` |
| 完了（緑） | `#3F8A78`（チェック背景）／補足文字も同色 |
| 選択中（青） | `#4A6FA5`（枠・チェック）／背景 `#F4F7FB`／淡背景 `#EDF2F9` |
| カウントダウン（柑子） | アイコン・数字 `#DD9627`／背景 `#FCF3E2`／文字 `#8A5E14` |

### タイポグラフィ
- 見出し・数字・ボタン：`Zen Maru Gothic`（丸ゴシック、太字700）
- 本文・補足：`Noto Sans JP`
- 日付ラベルは小さめ・グレー・字間広め。

### 形・余白
- カードの角丸：大きめ（14〜16px）、外周コンテナは 24px。
- ピル（たのしみ）：角丸 999px。
- 完了カードは opacity 0.75 で沈める。
- 全体に余白を広めに取り、詰め込まない。

### アイコン（Tabler Icons）
モード等の固定アイコン：
- じぶんで選ぶ `ti-hand-finger`／気分に合わせる `ti-cloud`
- 軽めの日 `ti-feather`／ふつう `ti-walk`／しっかり `ti-flame`
- 完了チェック `ti-check`／交換 `ti-switch-horizontal`／ロック `ti-lock`
- カウントダウン（旅行）`ti-plane-departure`

たのしみ・まいにちのアイコンは name から自動割り当て（対応表）：
| キーワード | アイコン |
|---|---|
| ボルダリング・登山・運動 | `ti-mountain` |
| 漫画・本・読書 | `ti-book` |
| アニメ・動画・TV | `ti-device-tv` |
| ゲーム | `ti-device-gamepad-2` |
| 音楽 | `ti-music` |
| 映画 | `ti-movie` |
| 散歩 | `ti-walk` |
| 該当なし（デフォルト） | `ti-sparkles` |

イベントのアイコンも同様に name から推測してよい（旅行→`ti-plane-departure`、誕生日→`ti-cake` など。デフォルトは `ti-calendar-heart`）。

## 8. CSVパーサの注意

- 1行目はヘッダーとして列名を取得。
- 値の前後の空白はトリム。
- 空行は無視。
- 値にカンマや改行を含むことは想定しない（含めたい場合のみダブルクオート対応を足す）。まずはシンプルな実装でよい。
- `energy` 空欄は `ふつう` に補完。`days` 空欄や `毎日` は全曜日扱い。

## 9. 公開（GitHub Pages）

最終的に次の構成をリポジトリ `kyou-no-3` 直下に置く：

```
kyou-no-3/
├─ index.html
├─ routines.csv
├─ projects.csv
├─ events.csv
├─ fun.csv
├─ daily.csv
└─ README.md
```

### Claude Code に期待する手順
1. 上記ファイルをローカルに生成。
2. `git init` → コミット。
3. `gh repo create kyou-no-3 --public --source=. --push`（`gh` CLI 認証済み前提）。
4. GitHub Pages を有効化：`gh api -X POST repos/{owner}/kyou-no-3/pages -f "source[branch]=main" -f "source[path]=/"`（または Settings→Pages 相当の操作）。
5. 公開URL（`https://<ユーザー名>.github.io/kyou-no-3/`）を表示。反映には数十秒〜数分かかることを伝える。

### README に含める内容
- アプリの概要と公開URL。
- **日々の更新方法**：`projects.csv` の `next` 列を、次の一手が終わるたびに書き換える（GitHubのファイル画面で鉛筆アイコン→編集→コミット、スマホからも可）。変更は数十秒でPagesに反映され、アプリを開き直すと最新になる。
- 各CSVの列の意味（本仕様書の第2章の要約）。
