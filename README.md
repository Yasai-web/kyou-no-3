# きょうの3つ

毎日のタスク・楽しみ・予定をまとめて「見るとわくわくする」シンプルな Web アプリです。
やることを管理するためではなく、**「今日やることが軽やかに絞られて、見ると安心する」**ことを目的にしています。一日にメインで取り組むのは **3つだけ**。全部終わったら、何も残らない静かな達成画面になります。

**公開URL**: https://<ユーザー名>.github.io/kyou-no-3/

> ↑ GitHub Pages の公開後、`<ユーザー名>` をご自身のアカウント名に置き換えてください。反映には数十秒〜数分かかります。

スマホのブラウザで開き、**ホーム画面に追加**すると、ネイティブアプリのように使えます。

---

## 使い方

1. 朝、開くと「今日は、どう決める？」から始まります。
   - **じぶんで選ぶ**：候補リストから最大3つをタップ。
   - **気分に合わせる**：軽め／ふつう／しっかり を選ぶと、デイリー・ルーティン・プロジェクトから1つずつ、エネルギーに合う3つを自動で選びます。
2. 日中は「今日はこれだけ」画面。3つのタスクをタップで完了、交換アイコン（⇄）で1つだけ入れかえ、「残りを選び直す」でやり直しもできます。
   - 入れかえは **じぶんで選ぶ**（リストから手動）か、**気分に合わせる**（軽め／ふつう／しっかり を選ぶと、その1枠に合うものを自動で1つ選び直し）から選べます。気分での入れかえは、いまの3つとは必ず別の項目に変わります。
3. 3つ全部終わると「今日はおしまい」。隅の「もう1つやる？」で追加もできますが、システムから促すことはありません。

一日の境界は **早朝4:00**。深夜に作業していても4時までは「まだ今日」です。日付が変わると、その日の選択・完了チェックはリセットされます（過去に選んだ履歴だけは、同じものばかりにならないよう内部で保持します）。

状態（完了チェックなど）はブラウザの **localStorage** にのみ保存します。サーバーには何も送りません。

## 日々の更新方法（いちばん大事）

普段さわるのは **`projects.csv` の `next` 列だけ** です。

プロジェクト本体は大きすぎてタスクにできないので、**「次の一手」（next）に書いた小さな具体的アクションだけ**が今日の候補になります。次の一手が終わるたびに、`next` を次の小さな一歩に書き換えてください。

**スマホからの編集手順**：
1. GitHub でこのリポジトリの `projects.csv` を開く。
2. 鉛筆アイコン（Edit）をタップ。
3. `next` 列を書き換えて Commit（コミット）。
4. 変更は数十秒で Pages に反映されます。アプリを開き直すと最新になります（キャッシュ回避済み）。

## CSV ファイルの意味

すべて UTF-8・1行目はヘッダー・リポジトリ直下に置きます。

### `routines.csv` — 単発で繰り返すルーティン
週1・月1・不定期メンテナンスなど、毎日ではない定期タスクです。自動選択では、ここからまず1つ選びます。

| 列 | 内容 |
|---|---|
| `name` | タスク名（そのまま表示） |
| `min` | 所要時間（分・数値） |
| `days` | 候補に出す曜日。`日月火水木金土` を連記、または `毎日` |
| `energy` | エネルギータグ `軽い` / `ふつう` / `重い`（空欄は `ふつう`） |

### `daily.csv` — デイリー候補
「毎日必ず全部やるもの」ではなく、「今日のデイリー枠に出てもよい候補」です。自動選択では、ここからまず1つ選びます。

| 列 | 内容 |
|---|---|
| `name` | タスク名（そのまま表示） |
| `min` | 所要時間（分・数値） |
| `energy` | `軽い` / `ふつう` / `重い`（空欄は `ふつう`） |

### `projects.csv` — 継続的な取り組み（毎日さわる生きたファイル）
`next`（次の一手）に書いた小さなアクションだけが候補になります。曜日の概念はありません。

| 列 | 内容 |
|---|---|
| `name` | プロジェクト名（カードでは小さい補足として表示） |
| `next` | 次の一手（カードでは大きい主文として表示） |
| `energy` | `軽い` / `ふつう` / `重い`（空欄は `ふつう`） |

### `events.csv` — 楽しみなイベント（カウントダウン）
今日以降で最も近い1件を「あと◯日」と表示します。当日は「今日!」、過去は表示しません。

| 列 | 内容 |
|---|---|
| `name` | イベント名 |
| `date` | 日付（`YYYY-MM-DD`） |

### `fun.csv` — きょうのたのしみ（眺めるだけ・チェックなし）
毎日すべて表示。完了チェックは付きません（やってもやらなくてもいい楽しみ）。アイコンは名前から自動で割り当てます。

| 列 | 内容 |
|---|---|
| `name` | 楽しみの名前 |

### `everyday.csv` — まいにち（毎日固定の軽い習慣・チェックあり）
毎日必ず表示される軽い習慣。完了チェックあり。「今日の3つ」の自動選択には含めません（別枠で常に出るため）。少数に絞るのがおすすめです。

| 列 | 内容 |
|---|---|
| `name` | タスク名 |

## 技術構成

- 単一の `index.html`（HTML / CSS / JavaScript すべて内包）。ビルド工程なし。
- 外部依存は CDN のみ：Google Fonts（Zen Maru Gothic / Noto Sans JP）、Tabler Icons。
- データは同一リポジトリの6枚の CSV を `fetch` で読み込み（CORS 問題なし）。
- 「今日の状態」は localStorage に保存。サーバーサイドなし、GitHub Pages で静的公開。
- オフライン時は、前回読み込めた内容で動作を継続します。

---

## プライバシーに関する注意

このリポジトリを **public（公開）** で作ると、`projects.csv` などに書いたタスク名・日程が理論上は公開ウェブに載ります。気になる場合は、名前を略称にするか、リポジトリを private にしてください（private の場合、GitHub Pages の公開には条件があります）。
